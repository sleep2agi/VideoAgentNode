# 视频制作全流程 SOP

## 流程概览

```
选题 → 脚本 → AI素材生成 → 语音合成 → 视频合成 → 渲染 → QA → 发布
```

## 1. 选题

- 在 GitHub Issue 中创建选题，标注视频类型（宣传/资讯 or 知识点）
- 明确目标受众、时长、发布平台

## 2. 脚本撰写

- 确定视频结构：开场 → 正文（分段） → 结尾 CTA
- 每段标注：画面描述 + 旁白文字 + 时长预估
- 脚本模板见 templates/

### ⚠️ 首帧与前3秒铁律

**首帧就是封面！前3秒决定留存率！**

- 第一帧必须有视觉冲击力，不能是黑屏/空白/淡入
- 首帧要当封面缩略图设计：有主标题 + 有画面 + 有信息量
- 前3秒必须让观众知道"这个视频讲什么"且"想继续看"
- 参考高播放量视频：开头直接展示最终效果或抛出震撼数字

### 高播放量视频共性（必须内化）

1. **真实产品录屏 > 动画/PPT排版**（90%高播放技术视频是录屏）
2. **5-10分钟是B站爆款区间**（宣传片2分钟，教程5-8分钟）
3. **标题降门槛**：保姆级/小白/X分钟/一行命令
4. **封面设计**：深色背景 + 高对比大字 + 数字突出 + 不超过3个视觉元素
5. **一个核心概念**：不堆功能，只讲一个"啊哈时刻"
6. **开头展示最终效果**：让观众先看到"能做到什么"再讲"怎么做"

## 3. AI 素材生成（Dreamina）

- 根据脚本中的画面描述生成图片/视频片段
- 使用 text-to-image 或 image-to-video
- 小批量生成，验证质量后再批量扩展
- 注意 credits 余量：生成前 `dreamina user_credit` 检查

## 4. 语音合成（MiniMax TTS）

- 将旁白文字输入 MiniMax TTS 生成音频
- 选择合适的音色和语速
- 输出格式：WAV 或 MP3

## 5. 视频合成

### 方案选择（重要！）

| 场景 | 推荐方案 | 不推荐 |
|------|---------|--------|
| 产品宣传视频 | **真实录屏**（Playwright/OBS）| Remotion CSS 排版（像PPT） |
| 数据可视化/信息图 | Remotion | 纯录屏 |
| 教程/操作演示 | 录屏 + 极简文字标注 | 花里胡哨动画 |

### 录屏方案（产品宣传首选）

- Playwright recordVideo 录制产品实操（1920x1080）
- 关键操作慢放，无聊段落加速
- ffmpeg drawtext 叠极简文字标注（不超过10个）
- 让产品自己说话，像 Linear/Cursor/Devin 那样

### Remotion 方案（信息图/数据可视化）

- 使用 Remotion 编排素材：图片/视频 + 音频 + 字幕
- 配置分辨率、帧率、时长
- CJK 字体需显式指定路径，避免 □□□ 乱码
- ⚠️ 双列布局（左截图+右文字）本质是 PPT，产品宣传视频不要用

## 6. 渲染（ffmpeg）

- Remotion 输出帧序列 → ffmpeg 编码为最终视频
- 常用参数：`-c:v libx264 -preset medium -crf 23`
- 字幕渲染：`drawtext` 滤镜，注意 fontfile 路径

## 7. 质量检查（QA）

- `ffprobe` 验证输出文件：时长、分辨率、帧率、音轨
- 目视检查：画面完整性、字幕对齐、音画同步
- AI 自评不可靠，必须人工/工具验证

## 8. 发布

- 上传至目标平台（B站、YouTube 等）
- 填写标题、描述、标签
- 在 Issue 中标记完成，附发布链接
